# 页面路由规范 (Routing Specifications)

本规范定义了 Next.js 14 App Router 下的所有核心页面路径及其职责。它反映了当前系统能处理的所有导航流。

## 1. 首页 (`/`)
- **功能定位**: 作为品牌的形象窗口，以及引导用户进行 3D 生图定制的核心漏斗起点。
- **页面状态**: 已完成完备的多主题渲染构建。
- **关联规范**: 详见 `specs/frontend/themes.md` 以获悉所有共存的主题细节。

## 2. 定制流程中心页 (`/customize`)
- **功能定位**: 系统的核心工具链，用户将图转化为 3D 的全屏交互式工作场。
- **流程 Steps (业务要求)**:
  1. 上传原始单反/高分辨率图片 (Image Upload & Validation)。
  2. 调用并展示 3D 风格化模型的生成等待过程 (AI Generation Proxy 对接)。
  3. 提供全景预览与规格细微化配置（目前限定尺寸：`6cm`, `8cm`, `10cm`, `15cm`）。
  4. 确认后将其结构化 payload 推入客户端购物车 (Zustand 持久化)。

## 3. 商品列表展示页 (`/products`)
- **功能定位**: 常规预设商品或展示系统过往高度成功案例的陈列大厅。
- **数据来源**: 此处不同于完全从零定制，其数据流应当从被授权的 Shopify GraphQL Storefront API 中拉取已存在的通用 SKU。

## 4. 购物车与独立侧边栏 (`/cart` & `<CartSidebar>`)
- **展现范式**: 要求构建时必须提供两种视图体验：
  - 点击右上角的随手唤起：全局弹出的右侧划出 `Sidebar` 抽屉，方便用户不跳出当前体验。
  - 标准结算浏览：独立的全渲染 `/cart` URL 路由视图。
- **功能交互**: 并发增删改商品列表、修改购买份数，且自动侦听并与本地 localStorage 及 Zustand 同步。

## 5. 结账路由锚点 (`/checkout`)
- **功能归属**: 不在本项目内部域内进行实体 DOM 的渲染。
- **逻辑**: 作为逻辑中转站（Redirect Router）。组装好当前购物车的所有定制及附加费用 Payload 后，重定向抛转至独立的 Shopify Hosted Web Checkout URL。
